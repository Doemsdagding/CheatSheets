ChromaDB ‚Äì INTRODUCTION

ChromaDB
ChromaDB is an open-source vector database specifically designed for storing, managing, and querying vector embeddings and their associated metadata.
ChromaDB is primarily used to enable semantic search, where the system understands the meaning and context of a query, rather than just keyword matching, and for powering AI applications, especially those involving Large Language Models (LLMs), like Retrieval-Augmented Generation (RAG), recommendation systems, and AI-powered chatbots.

Vector embeddings
Vector embeddings are numerical representations of data (like text or images) that capture their semantic meaning, allowing similar data to be located near each other in a high-dimensional space

Collection
WHile a colleciton is similar to a sql table its not the exact same.

SQL Tables are for when you need to ask what data meets a precise, structured criteria (e.g., "Find all customers in London who bought a laptop in 2024").

ChromaDB Collections are for when you need to ask which data is semantically similar to a query (e.g., "Find all documents about baking that are like my query about making cakes")

Where operators
When queriing you can use a straight dictionary in your query   for the where clause, this will use to defautl equal to operator.

But you can also by adding a dictionary containiing an operator and the original value as key value pair, change the operator.

Operator

Use
$eq
 equal to (string,int,float)
$ne
 not equel to (string, int, float)
$gt
 greater than (int, float)
$gte
 greater than or equal to ( int, float)
$lt
 less than (int, float)
$lte
 less than or equal to (int, float)

Example
# default equal to
result = collecction.query(
    query_texts = {"movies where poeple sing a lot"],
    n_results = 3
   where = {"type": "movie"}
)
print(result)

# indicatedequal to
result = collecction.query(
    query_texts = {"movies where poeple sing a lot"],
    n_results = 3
   where = {"type": { "$eq": "movie"}
     }
)
print(result)
Where filters can be conbined with logical operators this is achieved by starting the dictionary with the logical operator and than as a list of dictionaries giving the clauses

Logical operator

Use
$and
 and
$or
 or

Example
result = collecction.query(
    query_texts = {"movies where poeple sing a lot"],
    n_results = 3
   where = { "$and": [ {"type": { "$q": "movie"}},
             {"release_year":{"$gt": 202} }
            ] }
)
print(result)
