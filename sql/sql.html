SQL– INTRODUCTION

SQL
sql can be used to query a sqldatabase which is a structured database which stores data in the form of tables ,rows and columns. sql has several flavors each slightly different but at the same too e.g. you have PostgreSQL used for prostgres database and T-SQL for microsoft sql server
tables
 table names should be in lower case and use _ instead of spaces.
columns
 column name should be in lower case and use _ instead of spaces and should unique and be written in singular rather than plural (if a space is used in t-sql use [<column name>] in postgresql use "column name"
schema
 schema references 2 thing the grouping of the tables by default "dbo" and the blueprint of the table it self containing the field names and its data types

Query
a query is used to retrieve data from a table it generally starts with "select" than column names or "*" for all column and then the "from" clause and joins to indicate from which table data should come and end with an ";"

Example
select *
from table_a

aliasing
to make column names unique or just to create more clearity while querying the "as" can be used after a field name to alias it in the result.

Example
select
name as employee_name
from table_a ;

distinct and top/limit
right after the "select" you can add additional keywords to manipulate the result by adding "distinct" the result will only return unique rows.
by adding "top <number>" the result will only containing the indicated amount of rows (or less if there are less).
(for t-sql top is used for postgresql limit is used

Example
select distinct years from table_a;
#t-sql
select top 10 years from table_a;
#postgresql
select years from table_a limit 10;

count()

count(*) can be used to count total records in a table rather than non empty values in a column

Example
select count(birthdate) as count_birthdates from people;
select count(*) as total_records from people;

order by
To sort the results the "order by" clause can be used by default it is set to "asc" but by adding "desc" behind it it will order in descending order.

Example
select title, budget
from films
order by budget, release_year desc;

group by
to aggreagte the data it self the "group by" clause can be used. any field that you select onwhich no aggregate function is applied should be in the group by clause when querying.

Example
select certification, count(title) as title_count
from films
group by certification;

filtering groups
while you pre filter the data using a "where" clause before the group by is applied in order to filter the group it self the "having" clause is used. it works the same way but then you use having instead of where.

Example
select
release_year, count(title) as title_count
from films
group by release_year
having count(title) > 10;

set operations
sql has 3 set operations being "union", "intersect" "except" used to combine the results from multiple queries. it requires the same amount of columns for each query as it doesn't add columns just rows together

union
the "union" set operator is used to combine the results of the queries keeping all records (expect duplicates ) of both queries

Example
select * from left_table
union
select * from right_table;

union all
the "union all" is the same as union but where union removes the duplicate rows and the union al keeps the duplicate rows.

Example
select * from left_table
union all
select * from right_table;

intersect
intersect takes two results as input and returns only the records that exist in both results

Example
select id, val from left_table
intersect
select id, val from right_table

Except
except allows to filter the first result to keep only the records that are unique to the first result and dont exist in the second result

Example
select monarch,country from monarchs
except
select prime_minister, country from prime_ministers;

joins
often more than one table is required for a query for this joins are used.

inner join
The "inner join <table> as <alias> on <column> <comparison oerpator> <column>"  joins the two tables but only returns the records of the left table of which the on statement resturns true ( can create duplicate rows)  in postgres when the column name is the same the using clause can be used

Example
# t-sql and psotgres
select *
from presidents as p1
inner join prime_ministers as p2 on p1.country = p2.country

#postgress
select *
from presidents as p1
inner join prime_ministers
using(country);

left join
The "inner join <table> as <alias> on <column> <comparison oerpator> <column>" returns all records of the left table no matter if the on statement is true or not. (can create duplicate rows) in postgres when the column name is the same the using clause can be used

Example
select *
from presidents as p1
  left join prime_ministers as p2 on p1.country = p2.country

right join
right join does the reverse of left join keeping all the records of the right table and adding the data from the left table to it

Example
select *
from presidents as p1
  right join prime_ministers as p2 on p1.country = p2.country

full join
full join is left and right join combined resulting in that all data from both left and right tables are combined into one table .

Example
select *
from presidents as p1
  full join prime_ministers as p2 on p1.country = p2.country

cross join
cross join doesnt have an on clause as it joins every combination with every combination.

Example
select id1, id2
from table1
cross join table2;

Nested queries
there are several methods to apply nested quries inside quries.

inside in operator (and not in)
instead of providing the in clause with a list you can provide with a column instead supplied by a query.  to filter on just the result of the query.

Example
select country, president
from presidents
where country not in (
select country from states )

outer apply
an outer apply is used to nest a query inside a join but the query will be ran for each row seeratly in a loop. its generally used in combination with top to only add 1 specific data set to a row its acts as a left join.

Example
SELECT
    d.DepartmentName,
    e.EmployeeName,
    e.HireDate
FROM Departments d
OUTER APPLY (
    SELECT TOP 1 EmployeeName, HireDate
    FROM Employees
    WHERE DepartmentId = d.DepartmentId
    ORDER BY HireDate DESC
) AS e;

cross apply
cross apply is the same as an outer apply but then it acts like an inner join

Example
SELECT
    d.DepartmentName,
    e.EmployeeName,
    e.HireDate
FROM Departments d
CROSS APPLY (
    SELECT TOP 1 EmployeeName, HireDate
    FROM Employees
    WHERE DepartmentId = d.DepartmentId
    ORDER BY HireDate DESC
) AS e;

exists
exsits can be added to the where clause to filter the table on the existence of a connection without adding the data to the original table.

Example
SELECT
    d.DepartmentName
FROM Departments d
WHERE EXISTS (
    SELECT 1
    FROM Employees e
    WHERE e.DepartmentId = d.DepartmentId
      AND e.HireDate >= '2024-01-01'
      AND e.HireDate <= '2024-12-31'
);

inside select, where and from

by using brackets and an alias you can put sub queries inside a select statement. unlike the outer apply the nested query is executed all together and stored in work memory rather than executing it for each row.

Example
select distinct continent,  ( select count(*) from monarchs where states.continent = monarch.continent) as monarch_count
from states;

SELECT * FROM populations
WHERE year = 2015  AND life_expectancy > 1.15 *   (SELECT AVG(life_expectancy)
   FROM populations  WHERE year = 2015)  AND year = 2015;

SELECT local_name, sub.lang_num
FROM countries,
    (SELECT code, COUNT(*) AS lang_num
     FROM languages
     GROUP BY code) AS sub 
WHERE countries.code = sub.code
ORDER BY lang_num DESC;
