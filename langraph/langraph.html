LangGraph ‚Äì INTRODUCTION

LangGraph
LangGraph is an branch of the LangChain ecosystem. specifically for designing agentic systems, or systems including agents.

Agent
Agents use llms to take actions.AI agents exhibit a degree of autonomy, memory. making it a software system that uses artificial intelligence to autonomously pursue goals and complete complex, multistep tasks on behalf of a user or system.

Tools
Agents often use tools.Tools are functions called by the agent (like run code, load document, call upon an api ect).

ReAct agent
ReAct stand for reasoning and acting. a ReAct agent prompts the model using a repeated loop of thinking, acting, and observing. e.g. asking the weather to an agent it first interpertrs the questions than use its tool to call for the weather api, than observers the results and then gives the awnser.

besides making your own agents provide langgraph a variaty of prebuilt agents among which a ReAct agent in the form of the create_react_agent() function.
the create_react_agents() requires variables 1 containing the llm and the tools to be used

The langchain community allready made toolkits for a variety of pruposes among which doing math, these can be imported using the load_tools() function which takes 2 variables a list of tools and the llm to use these tools.

just like chains agents can be executed using the .invoke() function. the invoke methord returns a dictonary containing the key value pair messages and a list of messages that are retruned among wihich the input (humanMessage), the ais translation of the message to the tool (AiMessage), the tools response (ToolMessage) and the final output message of the ai (AiMessage) to
retrieve the response messages["messages'][-1].content) can be used to retrieve the content of the last message (aka the Ai response Message)

Custom tools
Tools in LangChain must be formatted in a specific way to be compatible with agents. They must have:
* a name, accessible via the .name attribute. ( by default this is the function name)
* a description, which is used by the LLM to determine when to call the tool. For example In case the of a math tool, if the LLM interprets the task as being a math problem, it will likely call this tool based on its description. (by default this is the docstring)
* return_direct paramater which indicates whether the agent should stop after invoking this too (set to default by default)

you can also print the argument names which shows the names an expected datatypes using the .args attribute

To turn a python function into a tool the @tool decorator is used

Example
