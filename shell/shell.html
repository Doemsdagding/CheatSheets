Shell
shell is accessed using a command-line terminal interface like alacritty. ach time a command is entered, the shell runs some other programs, prints their output in human-readable form, and then displays a prompt to signal that it's ready to accept the next command

Cancel a command
while a command is running press CTRL + C to cancel it.

-y
adding -y to a command makes it so that the terminal automatically respond yes to all prompts from the command

Directory navigation and manipulation

pwd
pwd is used to retrieve the current directory.

Example
pwd #/home/repl

ls
ls is used to list what is inside a directory, if not given a directory it list the current directory its contents
in some shell you need to use ls . to get current directory.

when not starting with / it will use a relative path when starting with a / it will use an abosulate path.

adding a -R flag will result in ls also displaying the underlaying direcotries their files

adding -F flag will add an / after directory and a * after an executable

Example
ls #backup bin course.txt  seasonal
ls seasonal
#autumn.csv spring.csv summer.csv winter.csv
ls /home/repl/seasonal
#autumn.csv spring.csv summer.csv winter.csv
ls -R -F   # seasonal/ course.text
./seasonal:
autumn.csv spring.csv asummer.csv

cd
to move between directories cd is used
by doing cd .. you move one directory above
using cd ~ brings you back to the home directory

Example
cd /home/repl
cd seasonal
cd ..
cd ~

cp
cp is used to copy paste a file by cp <source path><file.extention> <target path> 
by also adding a filename to the target you can rename the file durring the copy
cp <source path><file.extention> <target path><file.extention>

you can also use multiple source files and copy them to the same target directory
cp <file.extention> <file.extention> <target path>

Example
cp original.txt duplicate.txt
cp seasonal/autumn.csv seasonal/winter.csv backup

Files in shell 2: grep
grep selects lines according to what they contain it acts as head but than with a filter. by using 1 or more flags different filters can be used. without any flags it just prints all the line that contains the search word.

Example
grep molar seasonal/autumn.csv
#2017-02-01,molar
#2017-02-25,molar

-c
print a count of matching lines rather than the lines themselves

Example
grep -c incisor seasonal/autumn.csv seasonal/winter.csv
#seasonal/autum.csv:3
#seasonal/winter.csv:6

-h
do not print the names of files when searching multiple files. returning it like both are 1 file together

example
grep -h molar seasonal/autumn.csv seasonal/winter.csv
#2017-02-01,molar
#2017-02-25,molar
#2017-10-01,molar
#2017-10-25,molar

-i
ignore case (e.g., treat "Regression" and "regression" as matches)

example
grep -i molar seasonal/autumn.csv
#2017-02-01,molar
#2017-02-25,molar
#2017-02-26,Molar

-l
print the names of files that contain matches, not the matches

example
grep -l molar seasonal/autumn.csv seasonal/winter.csv
#seasonal.autum.csv
#seasoonal.winter.csv

-n
print line numbers for matching lines

example
grep -n molar seasonal/autumn.csv
#1:2017-02-01,molar
#5:2017-02-25,molar

-v
invert the match, i.e., only show lines that don't match

example
grep -v molar seasonal/autumn.csv
#2017-02-01,canine
#2017-02-25,wisdom

Files in shell 3: storing output
to store the out put of a head,cut ect in a file insteado of printing it in the terminal the ">" is used if the command started with the > it will still output the files just as when used in the middle (using it in the middle and writing the file after it is best practise)

Example
head -n 5 seasonal/summer.csv > top.csv

> result.txt head -n 3 seasonal/winter.csv

Running multiple commands in one line
to run multiple commands in one line and using the output of one command as the input for the next command the "|" is used. this can be done as many times as wanted.

Example
cut -d , -f 2 seasonal/summer.csv | grep -v Tooth
#Cainen
#canine
#wisdom

cut -d , -f 2 seasonal/summer.csv | grep -v Tooth | head -n 1
#Cainen

Files in shell 5: wildcards

*
To run a command normally with multiple files there are two options  using " " and writing each file name seperate

or using "foldername/*" to apply it to all files in a folder.

by adding an file extention or any letters after the * you can filter it further to e.g all csv files in a folder or ends with

by adding anything before the * you can filter on starts with

Example
cut -d , -f 1 seasonal/winter.csv seasonal/spring.csv seasonal/summer.csv seasonal/autumn.csv

cut -d , -f 1 seasonal/*

cut -d , -f 1 seasonal/*.csv

cut -d , -f 1 seasonal/s*

?
the "?" wild card is used to create a match question where single chracters could be that "?" e.g 201?.txt matches with 2010.txt, 2011.txt ec

Example
cut -d, -f 1 sales/sales_year_201?

[]
to make the "?" wildcard more specific "[..]" can be used anything written within the brackets will be seen as an option so [12] only matches the values 1 and 2.

Example
cut -d, -f 1 sales/sales_year_201[12]

{}
if you want the match to be more than one charachter "{,}"can be used. comma seperated anything written with then curly brakcets is checked durring the match check. eg {2010,2011} will only match with 2010 and 2011

Example
cut -d, -f 1 sales/sales_year_{2010,2011}

Files in shell 6: sort
using the "sort" command the output rows are sorted in order (by default ascending alphabetical order) using the flags
-n is used to sort numerically
-r is used to sort in reverse order
-b is used to ignore leading spaces
-f is used to sort case insensitive

Example
cut -d , -f 2 seasonal/winter.csv | grep -v Tooth | sort -r

uniq
when dealing with data you often find duplicates using "uniq" these duplicates can be filetered out. the downside of uniq is that you need to order the data first as it only removes adjecent duplicates so without sort it is less strong.

Example
cut -d , -f 2 seasonal/winter.csv | grep -v Tooth | sort | uniq -c
