Object oriented programming
OOP involves - thinking about objects and patterns of their interactions. It helps us organize our code better, making it more reusable and maintainable.
object =  a combination of data and functionality. For example, an object representing a customer can have data such as a phone number and email associated with them, and functionality like the ability to place or cancel an order. In Python, everything is an object. Numbers, strings, DataFrames, and even functions are objects
state = data associated with an object in python this is called attributes, attributes can be retrieved through the attributate method of <object>.<attribute_name> e.g. dataframe.shape
behavior = the functionality of an object, in python these are called methods. methods can be applied to an object through the method calling method <object>.<method_name>() distinigueshed by the (). e.g. dataframe.head()
class = are like blueprints for objects. They describe the possible states and behaviors that every object from that class can have. The unified templates that classes create are what allow Python objects of the same type to behave in the same way. all attributes and method can be retrieved using dir(<object/class>)

Class-level attributes
a class level attribute is an attribute that is the same for all class object  by giving it a fixed value. when setting a class level attribute you dont set it using self. when using the attribute in th class function rather than self. the class name is used.
* the convention is to capatalise class level attributes

after creating the class object it is still possible to overwrite the class_level attributes but it will only overide for that class object.
class level attributes can be used for avoiding repetition of setting min and max values to check against durring setting of attributes.

Example
class Employee:
    #define a class attribute without self
    MIN_SALARY = 3000
   
    def __init__(self, name, salary):
         self.name = name
         if salary >= Employee.MIN_SALARY:
               self.salary = salary
          else:
                 self.salary = Employee.MIN_SALARY

emp1 = Employee("sabra", 40000)
print(emp1.MIN_SALARY) #3000

emp1.MIN_SALARY = 5000
print(emp1.MIN_SALARY) # 5000

class inheritance
class inheritance allows the reuse of previously made code and add or modify functionaly.

inheritance is achieved through creating a new class but adding the parent class as an argument.
giving just the word pass in the class result in the child class inherting everything 1 on 1 from the parent including its constructor.

Example
class BankAccount:
    def __init__(self, balance):
          self.balance = balance

    def withdraw(self, amount):
           self.balance -= amount

# class inherting from baankaccount
class SavingsAccount(bankAccount):
    pass

savings_acct = SavingsAccount(1000)

customizing constructors
adding a constructor to a child class overwrite the parents constructor class.
by adding the parents constructor class inside the childs constructor class you can still inherit the parents constructor method while given the option to add additional arguments to the childs class constructor
furthermore you can add new functionality in the form of methods that use both the parents arguments and the childs added arguments. .

Example
class SavingsAccount(bankAccount):
    def __init__(self, balance, interest_rate):
         #inherit back the init constructor from theparent class
         BankAccount.__init__(self, balance)
        
           #add its own additional argument
           self.interest_rate = interest_rate

            #new functionality
            def compute_interest(self, n_periods=1):
                  return self.balance * ( (1 + self.interest_rate) ** n_periods - 1 )

# create a savingsaccount
acct = SavingsAccount(1000, 0.03)
