Building Ui: Introduction
In flet the UI is composed of widgets known as controls each being the child of a parent class except for page. each control has its on set of unique methods and functions reffered to as properties (some are shared some are unique to the control)

the base of the application is the main() functions which contains the primary logic of the application.
the page parameter serves as the parent control for all other controls.
controls can be added, removed or manipulated within the page, using methods such as add, insert, remove.

additionally the page control can contain nested pages. by default the page control applies acolumn layout from top to bottom.

FLET objects 1: Page
The base of a flet app is the page. when creating a new app you start with a page, the page can contain other objects(also called controls) and other pages.
by default the Column layout is used (used to add objects in a single column top-to-down.

Typically the page is defined in the def main.

Example
import flet

def main(page: flet.Page):
  page.title = "test"
  page.bgcolor = flet.colors/WHITE
 
  text = flet.Text("hello world")
  page.add(text)

Flet.run(main)
A page can have many attributes the most common are:

page.title
The page.title sets the name of the application. it takes a string(in desktop app program title)

Example
page.title = "my app"

the page.bgcolor allwos to set of a page is background color it takes a named color or a hex code

page.add()/ page.insert()

There are multiple methods to add other objects to a page. this can be done using "page.add(<object>)"

To add an object to an existing list of page object at a particular position (specified by index) the "page.insert(<index>,<object>") is used.

Example
   text = flet.text("hello world")
   page.add(text)
   page.insert(2, text)

page.remove()/ page.remove_at() / page.clean()
When a object is not needed anymore to be shown on the screen the "page.remove(<object>)" can be used. To remove an object with a specific index from the list of added controls "page.remove_at(<index>)" is used.
if you want to remove all objects from a page you can use the "page.clean()" the app returns then to a blank page.

Example
page.remove(text)
page.remove_at(2)

page.clean()

page.scroll
The page.scroll property allows for vertical scrolling to prevent page content (controls) from overflowing. There are five ScrollMode options available:
    None (default): The row is non-scrollable, and its content could overflow.
    AUTO: Scrolling is enabled, and the scrollbar is only shown when scrolling occurs.
    ADAPTIVE: Scrolling is enabled, and the scrollbar is always shown when running the app as web or desktop.
    ALWAYS: Scrolling is enabled, and the scrollbar is always shown.
    HIDDEN: Scrolling is enabled, but the scrollbar is always hidden

on_scroll (column & row)
the on_scroll event fires when the scroll position is altered by a user.
The Column and or row receives an instance of the flet.OnScrollEvent class with the following properties:

event_type: Describes the type of scroll event (start, update, end, user, over).
    start ‚Äì control has started scrolling.
    update ‚Äì control has changed its scroll position.
    end ‚Äì control has stopped scrolling.
    user ‚Äì user has changed the direction in which they are scrolling.
    over ‚Äì control has not changed its scroll position because the change would have caused its scroll position to go outside its scroll bounds.
pixels: Current scroll position in logical pixels.
min_scroll_extent: Minimum in-range value for pixels.
max_scroll_extent: Maximum in-range value for pixels.
viewport_dimension: Extent of the viewport.
scroll_delta: Distance scrolled by the scrollable, set for update events.
direction: Direction of user scrolling (idle, forward, reverse), set for user events.
overscroll: Number of logical pixels the scrollable avoided scrolling, set for over events.
velocity: Velocity at which the ScrollPosition was changing when overscroll happened, set for over events.

Example
flet.Column(

 on_scroll = flet.OnScrollEvent(
            t=5,
            p=2,
            minse=10,
            maxse= 10,
            vd=10,
            sd=5,
            dir=1,
            os=6,
            v=20,
        )
)

Flet Controls offer various properties and methods for customization, some unique to specific controls while others are shared among all. Common methods are uniformly applied across controls. Here are some common methods and their application approach.

In Flet, a key is an identifier for a Control, allowing Flet to uniquely identify and track them. Keys are used for control state management, efficient control updates, and handling rendering.

In Flet ref is reference to a specific control or element within the UI hierarchy.
