Setting up a desktop app
while setting up web and or phone app is a bit more complicated setting up a desktop app is easy by just defining a page

Example
import flet

def main(page: flet.Page):
    pass

flet.app(target = main)

setting up a mobile app
to create a mobile application a slight different method should be used as there are additional dependencies for this purpose the best option is to use a terminal command in an empty folder (see example)

this command will generate  a project structure including:
* assets/: = a directory for media files such as images, sound  (if only icon.png is provided it will be used as source image to genree icons and splash screens for all platforms)
* main.py = the file that serves as the entry point of the application
* requirements.txt = file containing al lteh depencies requried for creating the flet app (if file is not provided only flet dependencys are installed durring packaiging)
* pyproject.toml =  file meatn for readin dependencies.

Example
flet create <project_name>

For mobile:

Building Ui: Introduction
In flet the UI is composed of widgets known as controls each being the child of a parent class except for page. each control has its on set of unique methods and functions reffered to as properties (some are shared some are unique to the control)

the base of the application is the main() functions which contains the primary logic of the application.
the page parameter serves as the parent control for all other controls.
controls can be added, removed or manipulated within the page, using methods such as add, insert, remove.

additionally the page control can contain nested pages. by default the page control applies acolumn layout from top to bottom.

Packaging and publishing apps

flet pack (desktop: mac,windows, linux)
for desktop flet has an Pyinstaller integration which can make executables compatible for windows, macos and linux.

in order for an unique icon the pillow library needs to be installed (pip install pillow)

to initiate go into the project directory and use the command.
 "flet pack main.py --name <app name>  --icon <path to icon>"

the executable for each desktop env can be found in the dist folder.

flet publish: (static websit)

for static website use:
"flet publish main.py

flet build: (mobile, website, desktop )
flet build supersedes both publish and pack. instead of pyinstaller it uses the Flutter SDK. for this reason Flutter SDK needs to be installed and the bin directory needs to be added to the systems path environment.and than in a terminal run "flutter doctor"(onlinux there is also the requirment fro Gstreamer to be installed ( apt install libgtk-3-dev libgstreamer1.0-dev libgstreamer-plugins-base1.0-dev)

to build for a system your os is important to build apple you need to be on macos, to build for windows you need to be on windows to build for android, linux or web any os will do.

Building android APK
Use " flet build apk <path/to/app folder>"
(the folder path is optionally if your in the path allready with your terminal).to generate an Android APK file from your app. This command builds the release version, which doesn't support debugging and is suitable for deployment to app stores.

if wanted to publish to the google play store using Android App bundles (AAb) is recommended to keep file size small. achieved through the "flet build aab <path/to/app>"

building apple ipa
On a macOS host, you can use  flet build ipa or  flet build ipa /path/to/app to build an iOS archive bundle and IPA file for distribution. To generate the IPA successfully, ensure you provide correct values for the following arguments:

    --org: Organization name in reverse domain name notation (e.g., com.mycompany). It's combined with --project to form the iOS and Android bundle ID. Default is com.flet.
    --project: Project name in C-style identifier format (lowercase alphanumerics with underscores). Used to build bundle ID and as a name for bundle executable. By default, it's the name of the Flet app directory.
    --team: Team ID to locate the provisioning profile. If no team ID is provided, an unsigned iOS archive will be generated

You can customize app icons for all platforms (excluding Linux) using images in the assets directory of your Flet app.

iOS: Use icon_ios.png (or any supported image format). Recommended minimum size is 1024 px. (defautls to icon.png with alpha-channel automatically removed)

Android: Utilize icon_android.png (or any supported image format). Recommended minimum size is 192 px. (defaults t icon.png)

Web: Employ icon_web.png (or any supported image format). Recommended minimum size is 512 px. Defaults to icon.png.

Windows: Use icon_windows.png (or any supported image format). An ICO file of 256 px size will be produced. Defaults to icon.png. If icon_windows.ico file is provided, it will be copied to windows/runner/resources/app_icon.ico unmodified.

macOS: Utilize icon_macos.png (or any supported image format). Recommended minimum size is 1024 px. Defaults to icon.png.

FLET objects 1: Page
The base of a flet app is the page. when creating a new app you start with a page, the page can contain other objects(also called controls) and other pages.
by default the Column layout is used (used to add objects in a single column top-to-down.

Typically the page is defined in the def main.

Example
import flet

def main(page: flet.Page):
  page.title = "test"
  page.bgcolor = flet.colors/WHITE
 
  text = flet.Text("hello world")
  page.update()

Flet.app(target = main)
A page can have many attributes the most common are:

page.title
The page.title sets the name of the application. it takes a string(in desktop app program title)

Example
page.title = "my app"

page.bgcolor
the page.bgcolor allwos to set of a page is background color it takes a named color or a hex code

Example
page.bgcolor = flet.colors.Blue
page.bgcolor = "#ff0000"

page.add()/ page.add_async()/ page.insert() / page.update()
There are multiple methods to add other objects to a page. this can be done using "page.add(<object>)" or "page.add_async(<object>)" difference between the two is that the "add" adds them in the arrangemetn and priority set by the deverloper and that the "add_async" add them asynchronously.
To add an object to an existing list of page object at a particular position (specified by index) the "page.insert(<index>,<object>") is used.

after adding a object "page.update()" is used to refresh the app and show the added objects. often its best to batch any made changes to do one big update.

Example
   text = flet.text("hello world")
   page.add(text)
   page.add_async(text)
   page.insert(2, text)
page.update()

page.remove()/ page.remove_at() / page.clean()
When a object is not needed anymore to be shown on the screen the "page.remove(<object>)" can be used. To remove an object with a specific index from the list of added controls "page.remove_at(<index>)" is used.
if you want to remove all objects from a page you can use the "page.clean()" the app returns then to a blank page.

Example
page.remove(text)
page.remove_at(2)
page.update()

page.clean()
page.update()
