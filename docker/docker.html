Virtualization
Virtualization is the process of creating a virtual version of a computing resource.
A virtual machine is called full virtualization.Other levels of virtualization include virtualizing a specific computer component, such as the operating system or storage, which does not result in a virtual machine. often done through vmware.
with environment everything surrounding an it application is meant including software, hardware and the operating system
containers also know as OS-level virtualization isolates user spaces including application dependingsies and application settings, it virtalizes the OS but not the os kernel and hardware. resulting in that a single host can run multiple applications without requiring a dedicated OS for each application.

for building and running containers docker is the common choice for automatically managign and coordianting multiple containers kubernetes is often used.

Docker
docker has several important components making up docker

Docker desktop
A gui of docker (not necessary but handy for basic commands)

Docker engine
accessed through the terminal used for all the docker commands (more than you can do through the gui).

Docker Daemon
runs the docker actions that apply to the docker objects

Docker objects
Docker objects are the actual containers and images.

Images
An image is like a recipe it contains instrutions on how to build a container. the image takes form in the form of a docker file.

Container
Whats build from the docker images and is the virtualisation layer that runs the programs

Docker registry
docker registries are centralized locations for storing and managign container images, e.g. docker hub

Orchestration
Container orchestration comes in when you have multiple containers. orchesjtration is the automated management and coordination of multiple containers. for this an orchestrator is used. to achieve this a config file is written which describes the desired autocome. with this config file the orchestrator will allocate cpu memory ect over the containers that need it

To achieve this you can use docker swarm mode or kubernetes.

Starting a container
to start a container from an image you can use the "run" command. by adding the flag:
-it = will run image but create an interactive shell inside the container to communicate with.
-d = will run the container in background so you continue using the terminal (otherwise you have to use "exit" to leave the shell)
--name <container name> = lets you set the container name

Example
docker run <image name>

#test image will just print helloworld and then shutdown
docker run hello-world

#running ubuntu
docker run -it ubuntu

List running containers
the "ps" command returns a table containing all the dockers that are running including their container id to identify them by.
the flags:
-f "name=<container_name>" = allowing the search of a specific container with the container name

Example
docker ps
#container_id image command create status ports names
14465626, postgres, "docker-entrpint..." 1 minute, up, 5432, posty

Stop containers

the "stop" command in combination with the container id or name is used to stop a container from running

Example
docker stop 14465626
docker stop posty

Container logs
To see the logs that are being created by the docker container you can use the the "logs" <container-id> command. this will print all the logs made by the container.

by providing it with the flags:
-f = allows you to print the logs in real-time.
Example
docker logs -f 132253545

Cleaning up (deleting a container)
Stopping a container will result in just the container being turned off to delete all its data so you say rebuild or just to make space
you cna use the "container rm <contianer_id/name>" command

Example
docker container rm 13354543451

pruning
to completely remove all stopped containers all at once you can use the "prune" command.

Example
docker container prune

Docker hub
Docker hub is a website where you can download user made images, for things like databases and variety of programs.

downloading image is also called pulling. and can be done through locker using the "pull <image_name>" command. it will automatically download the latest version by adding
":<version_name>" behind the name you can download a specific version

Example
docker pull ubuntu:22.04

seeing all pulled images
By using the command "images" you can see all images available on your machine. it will return the name, version, image_id, when its created and size.

Example
docker images
#repository tag image id created size
hello-world latest 14632531534 7months 9kb

Delete an image
to delete an image you can use "image rm <image-name/id" to completely remove the image. (you can only delete image of which there are no more containers running)

Example
docker image rm ubuntu

pruning
to remove all unused images at once you can use the "prune -a" command which will delete all unused images. without the -a it will only remove old versions of own made images.

Example
docker image prune -a
