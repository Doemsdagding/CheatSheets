OPENAI_API - introduction

OPENAI API
The OpenAI API allows the access to openai models through python using the openai library.

   Tokens
Tokes are a unit of one or more characters used by language models to encode and interpret text.

Each model has a price per token, the price can be different for input and output tokes

Chat roles

Roles

Explanation
system
 controls assistants behavior of what questions it should answer and to which degree of knowledge.
user
 instruct the assistant by asking the question or giving the task
assistant
 response to user instruction can be used to define the style and tone of answers made by the assistant

User role (making a request)
Making a request to an openai endpoint an api_key is required (can be retrieved from the opnai website and then used in the python code)

an example is the chat.completions endpoint being the general chatbot this one takes the variables model and messages and max_completition_takens.

* model being the used gpt model
* messages being a  list of doctrinaires containing the role of the request and the content of the request (aka the prompt)
*max_completion_tokes = is optional variable restricting how many tokens are allowed to be used by the request to output
*temperature = an optional variable which controls the randomness of the response with 0 make it deterministic and focused and the closer too 2 you get the more random and creative it gets (default is between 0.7 and 1) (if it would respond the same thing when you ask the same thing again))

Example
from openai import OpenAi

client = OpenAi(api_key = "*********************')

response = client.chat.completions.create(
    model = "gpt-40-mini",
    messages=[{"role": "user", "content": "what is the openai api?""}] m
    max_completion_tokens=5,
    temperature=1
)
print(response)                                                                                                                        

output
once execute the response contains an object with different attributes for accessing different information
id: a request id
choices: a list containing the response message and if it executed any functions calls ect
created: the moment of execution
model: the model used in the request
object: the used endpoint and system fingerprint
usage: how many tokens were used and on what with prompt_tokens being input tokens.

each can be accessed individually

Example
print(response.choices)

# or specifically the message
print(response.choices[0].message.content)

system role use (role playing + guardrails)
using the system role within the messages list you can give the ai assistant a role to roleplay (eg a python developer)  there by creating a chatbot for a specific purpose with a more specific and specified knowledge base..

Furthermore within the system role content value you can through prompting build guard rails against misuse of the chat bot
    you can tell to the chatbot what its purposes aren't and what he is forbidden from answer and how to respond if a user still tries to use the chatbot for that purpose. The guardrails should always be in the form of an if statement.

Example
from openai import OpenAi

client = OpenAi(api_key = "*********************')

response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role":"system",
                            "content":"""you are a python programming tutor who speak concisely.     #role play role
                                 if your asked for specific, real-world financial advice respond with:       #guardrail
                                    im sorry i am not allowed to provide financial advice
                            """},
                         {"role": "user",
                           "content": "What is the difference between mutable and immutable objects?"}]]
)
print(response.choices[0].message.content)
