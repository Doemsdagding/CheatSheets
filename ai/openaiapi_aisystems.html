Response format
The OpenAi api allows the indiciation of the output strcuture of the response of a prompt.

This can be done through specificing the response_format parameter of the resposne class. eg json_object will result in the response to be an json opbject as the message.content

Example
from openai impor OpenAi

client = OpenAi(api_key="********************"

response = client.chat.completions.create(
    model="gpt-40-mini",
    message= [
     {"role": "user",
      "content": "Please write down five trees with their scientific 
       names in json format."}
       ],
       response_format ={"type": "json_boject"}
)
print(response.choices[0].message.content)

Reducing tokens
When a ratelimit eror occcurs due to the fact the prompt being to large, what can be tried is reducing the token usage by making the prompt smaller.
To help with this the tiktoken library can be used which is the tokenizer library openai uses. by checken the len the number of tokens can be retrieved. (this can be used to set tokenlimits)

Example
import tiktoken
encoding = tiktoken.encoding_for_model("gpt-4o-mini")
prompt = "tokens can be full words or grouped word"
num_tokens = len(encoding.encode(prompt))

Error handeling
the open ai api can run into errors and will in that case n return a python error relating to the occured problem for example:
* Connection errors ( caused by connection issueseither on user or service side) e.g. InternalServerError, APIConnectionerror, APITimeoutError. General retrying after a few minutes will solve the problem.

* Resource limits error (caused by limits on frequence of request or size of prompt) e.g. ConflictError, RateLimitError. generally fixed by changing the prompt.

* Authentication error. (ccaused by invalid api key or token is invalid,expired) e.g. AuthenticationError. (to solve this the api key needs to be checked)

* Bad request errors ( caused by malformed or incomplete api request like one with a missing or invalid a paramter e.g. BadRequestError (to fix generaly the code has to be reviewed and checked with the documentation)

as their python erros you can use try catch method to handle the errors.

Retrying
If a call is being rate limited another way to re send the prompt after teh ratelimit error occrus thsi can be done using the tenacity python library.
using tenacity a retry decorator can be made. in wich using the wait paratemer it can be indiated how long the wait time between tries should be and using teh stop paratemer when to stop. using the stop_after_attempt function an specific amount of tries can be set.

Example
from tenacity import (retry, stop_after_attempt,w wait_random_exponential)

@retry(wait=wait_random_exponential(min=1, max=60), stop = stop_after_attempt(6))

Function calling
While giving the Openai API a resposne format ensures the format type of the ouput it doesnt ensure that the ouput is exactly formatted the same. Function calling can be used to generate consistent structured output.This ensure the output can be used for calling other apis or when a fixed structured information schema is needed like controlling devices.

This can be achieved through the tools paramater in the create class. The tool paramater takes a function_defintion in the form of a list of dictionaries.

the functiondefition requires multiple key value paris being:
*  type = for example a function (which is used when an own defined function needs to be called)
* function = on its own is a dictionary containing the name of the function, a short description and the parameters the function takes which again is a dicionary on its own. with each key inside the paramaters being an key that will bie in the outputed json schema.

when using tools the response is located under a differnt key in the response dictionary being response.choices[0].message.tool_calls[0].function.arguments) as you can have mutliple tools in a message the message.tool_calls is a list.

Example

Moderation
Moderation in the context of the OpenAI API refers to the process of analyzing input to determine if it contains any content that violates predefined policies or guidelines. OpenAI provides a moderations endpoint as part of its API. it is used to evaluate if the output. it checks for hate,harassment, self-harm sexual and violence.

to caclulate the probability for these catogiries the moderations.create class can be used which takes in paramter the input. from this for each category the probability can be retrieved.

Example
moderation_response = client.moderation.create( input= """ ...until someone dras an explodign kitten. when that happens, that person exploders. they are now dead. this process continues until... """

print(moderation_response.results[0].categories.violence)
