HUGGING FACE - INTRODUCTION 

Hugging face
hugging face is a platform where the ai community can access/ download, collaborate, and stay informed on the latest open-source models, datasets and applications.
Hugging face provides library for python to train models, deploy applications and explore the hugging face hub (where you can find all these things on)

Models
All the models found on the hub are opensource and from their model card you can get information like, its use, supported languages, and among other things its license.
Use this model button you can choose among other methods, vllm and  transformers wich will provide python code to loade the model for inference or training

Running models
to run a model there are 2 options. 1 local inference (running it on local hardware to run the computations be it own cumputer or cloud-based-development-environment) or 2 an inference provider via hugging face api (there are partner organisation that provide access to hihg pefromance machines)

Datasets
Datasets found on hugging face can be individually maniulated through sql queries from the hugging face hub before being used for training in python .

Transformers library
The hugging face transformers python library simplifies working with pre-trained models both for inference and training.

local inference
The pipeline class can be used for performing local inference for any models and tasks available on hugging face.

It requires 2 variables being the task (being its purpose (eg text generation, chramatical checking, text classification and category assignement ect ect)) and the model (being the ai model used written as uploader/ model name).


after which the defined class becomes a function on its own taking a prompt and returning the models output.
in the form of a dictionary the value being the returned output of the model.

to the function additional paramaters can be given like:
* max_new_tokens = which limits the response to the set amount of tokes
* num_return_sequences = which indicates the amount of responses requested thereby adding additional key value paris to the output dictonary

Example

from transformers import pipeline

gpt2_pipeline = pipeline(task="text-generation", model="openai-community/gpt2")

prompt = gpt2_pipeline("what if ai", max_new_tokens=10, num_return_sequences=2)

for result in prompt:
    print(result['generated_text'])
#what if ai had never existed?
#what if ai could be really smarter than us?

The dataset library allows us to access,download,use and share datasets using python.

Auto classes
auto classes are a more flexible way to load models,tokenizers and other ai components without manual setup.
they offer more control compared to pipelines.  (eg pipelines automatically retrieves the paired tokenizer while autoclasses you can choose your own)

so where pipelines can be used for experiments auto classes offer more cstomization.

Automodels
To download a model the AutoModel class is used there are specific functions for specific tasks (eg for text clasifiction the automodelforsequencelassifiction is used) aferwhich (how it is called you can see on the modelcard) the name of the model is given.

Example
from transformers import AutoModelForSequenceClassification

model = AutoModelForSequenceClassification.from_pretrained("distilbert-base-uncased-finetuned-sst-2-english"

Autotokenizers
To prepare input for use in a model a tokenizer is used (it is recommned to use the tokenizer that was paired with the model so it performs the same as durring its training, but it is not mandatory) the retrieval method is smiliar to the model retrieval method when the paired tokenizer is used.

after which to tokenize a prompt the tokenize() function is used which take the prompt and returns the tokens in a list.

Example
from transformers import AutoTokenizer

#retrieve paired tokenizer
tokenizer = AutoTokenizer.from_pretrained("distilbert-base-uncased-finetuned-sst-2-english")

#tokenize
tokens - tokenizer.tokenize("aiL heping robots think and humans overthink:)")
print(tokens)

Aut class 2: building a pipeline with autoclass
To create a pipeline using autoclasses both the model and tokenizer need to be combined using the pipeline() class by given it a task= (the intended task of the ai) the model= the donwloaded model using the fuction appropiate to the task and the tokenizer= the downloaded tokenizer

Example
from transformers import AutoModelForSequenceClassification, AutoTokenizer, pipeline

my_model = AutoModelForSequenceClassification.from_pretrained("distiblert-base-uncased-finetuned-sst-2-english")
my_tokenizer= AutoTokenizer.from_pretrained("distibert-base-uncased-finetuned-sst-2-english")

my_pipeline = pipeline(task="sentiment-analysis", model= my_model, tokenizer= my_tokenizer)
