Uvicorn
Python frameworks like djagon, flask, fastapi ect need a way to communicate with the internet for this Uvicorn can be used as the gateway interface. uviccorn is a ASGI asynchronous Server Gateway Interface.
ASGI allows frameworks to handle asynchronous I/O operations (async/await). When a request needs to wait for I/O, the worker process can pause that request and switch to efficiently handle other requests, maximizing the use of a single process.

Uvicorn handles all the low-level networking complexity:

  *  It listens on a specific host and port.
   * It parses the raw network data (HTTP request lines, headers, body) into the structured Scope dictionary and provides the Receive/Send functions.
  *  It calls your ASGI application and then takes the messages your application sends via the send callable and formats them back into raw HTTP responses to send over the network.

In simple terms: Uvicorn is the high-performance engine that runs the code created by ASGI frameworks like FastAPI.
to use Uvicorn first you need to have an ASGI App (e.g. Fastapi app ) with a main.py and async functions

Example
# main.py

from fastapi import FastAPI
import asyncio

# 1. Create the FastAPI application instance
# This 'app' object IS the ASGI callable that Uvicorn expects!
app = FastAPI()

@app.get("/")
def read_root():
    """A synchronous endpoint."""
    return {"message": "Hello, World from FastAPI!"}

@app.get("/async-task")
async def perform_async_task():
    """An asynchronous endpoint that simulates an I/O wait."""
    print("Starting simulated database query...")
    await asyncio.sleep(2)  # Simulate a 2-second I/O bound wait
    print("Simulated database query finished.")
    return {"status": "Complete", "delay": "2 seconds"}
next make sure you have uvicorn installed "pip install uvicorn" than to launch  Uvicorn in the terminal you can use
"uvicorn main:app --reload" 
* <main> referencing the main.py 
* <app> the FastAPI() app
* --reload is optional and just for development purposes and tell uvicorn to watch your files for changes and restar the server automaticaully when the files change.
*--workers <number of workers int> is another optional flag which tells uvicorn to start multiple instances of teh ASGI applicationand run them parallel (managed by a parten process manager lik uvicorn manager or an external manager lik Gunicorn) (max is tpe 2 to 4 workers per core) 


* --bind <ip>:<port> another optional flag lets you bind uviorn application to a specific port of your host
it will then respond something like
INFO:     Uvicorn running on http://127.0.0.1:8000 (Press CTRL+C to quit)
INFO:     Started reloader process [12345]
INFO:     Started server process [67890]
INFO:     Waiting for application startup.
INFO:     Application startup complete.
and using curl or terminal you can now do curl http://127.00.0.1:8000 which will respond #{"message": "Hello, World from FastAPI!"}

another way to launch Uvicorn is using programmatic launch (aka a python script) it works in a similar way as the terminal command.

Example
import uvicorn
from main import app # Assuming your FastAPI app is named 'app' in main.py

if __name__ == "__main__":
    uvicorn.run(
        "main:app",  # The target ASGI application
        host="0.0.0.0",
        port=8000,
        log_level="info",
        reload=True  # Can be set here too
    )

Logging (-log-level <log level>)
* --log-level <log level>
is another argument that can be given durring the launch of the Uvicorn server when given it will log any creatd logs from that level (default is info so any logger.debug calls wont show up in the Uvicorn logging) the log levels are in order (debug, info, warning,error, critical)

in your code you use the defualt logging library to set up the logging using the logging.getLogger() function and supplying it with "uvicorn.error" and than using th logger.<log level> in your code to do any logging.

the loggin will show up in the terminal your running the Uvicorn server in or ccan be trievied with "docker logs <container_name> " if it is running within a docker container.

Example
from fastapi import FastAPI
import logging

logger = logging.getLogger("uvicorn.error")

app = FastAPI()
logger.info("app is running")

@app.get("/")
async def main():
     logger.debug("GET /")
     return "ok"
