API

Web Apis communication over the internet using HTTP. meaning the client sens a request to a server and the server returns a message to the client.

The tree most common types of web apis are:
* Soap = focus on strict formal api design (enterprise applciations use this)
* REST = focus on simplicity and scalabilty (is the most common architecture)
*GrapQL = focus on flexibility and optimized for performance

e.g. for http:// 305.5th-ave.com :80 /unit/243 ?floor=77

*http:// = protocol = means of transporation to use
*305.5th-ave.com = domain = the address of the api server
*:88  = port = the entrance of teh api server
*/unit/243 = path = is the unit of the api server each resource has its own unique location of the uniton the server
*?floor=77 = query = contain additional instructions

Reading messages
Both request and response messages can be split in 3 individual parts

Start-line
in the request = the startline is refered to as the request line and contains the request type e.g. GET along with the path

in response message the startline = the status-line it contains a 3 digit number indicatign the status code and a status message.
the status cods are grouped in 5 categories indicated by the starting int
1XX = informational responses
2XX = successful responses
3XX = Redirection message
4XX = client error response
5XX = Server error response

Authentication
there several methods to add authentication to an api request

basic authentication
has a high ease of mplementation  but lacks in security as it sends passwords unencrypted over the itnernet to the server.

to use basic authentication an authorization key needs to be added to the header of a api request.
this header must contain a base64-encoded combination of the username and password.
or
you can add an additional argument called auth to the request takign a tuple containg the username and password without any encoding.

Example
requests.get("http://api.music-catalog.com", auth=("username", "password"))

Working with json
Most apis work with json. while dictionaries are similiar to json its not the same therefore the json libary can be used to
* encode a python object o a json string
* decode a json string (retruned by the api) to a python object

Example
import json
album = {"id": 42, "title":"back in black"}
#encode
string = json.dumps(album)
#decode
album = json.loads(string)

Example

Error handling

4XX Client errors (most common)
401 Unauthorized
 The requet lacks valid authenticiation credentials for the request resource. add the correct authentication or check if apikey/pw are correct.
404 not found
 indicates that the server ccannot find the resource that was requested. check the path url is correct and the query isnt to selective.
429 too many requests
 the client has sent to many requests in a given amount of time. change the code so it doesnt retry to quick or put a wait between your requests.

5XX Server errors (most common)
500 internal server error
 the server experience an unexpected issue which prevents it from responding.
502 bad gateway
 the api server could not successfully reach another server it needed to complete the response
504 gateway timeout
 the server (which acts as a gateway) did not get a response from the upstream server in time
